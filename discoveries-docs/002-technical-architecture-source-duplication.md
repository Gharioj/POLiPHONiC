# Constitutional Discovery 002

## Discovery

The Technical Architecture's source `.docx` contains sections 5–7 twice, in two differently-formatted occurrences. This was judged a document-production artefact rather than a constitutional matter, and the later occurrence was kept in the integrated document — but the reason the source file was left in this state, and whether the source itself should be corrected, remains unresolved.

## Trail

- **First identified**, integrating the Technical Architecture (v0.4.0): "a documented duplication found in the source (sections 5–7 appeared twice; the later, differently-formatted occurrence was kept, both are quoted in the provenance record)." — [`CHANGELOG.md`](../CHANGELOG.md), v0.4.0
- Provenance record: "The duplication itself is treated here as a document-production artefact, not a constitutional question — but *why* the document was left in this state (an incomplete edit, a merge error, or something else) is not known, and the underlying source file has not been corrected. If the Owner intends this repository's Foundation to track a 'cleaned' master, that master should eventually be reconciled with the original at its source." — [`technical-architecture.provenance.md`](../constitutional-foundation/technical-architecture.provenance.md)
- **Later recognised as the first instance of a pattern**: see [Discovery 004](004-source-documents-found-incomplete.md), recorded when two further source documents were found incomplete (a distinct, more severe defect — missing content rather than duplicated content). The Interpretation Framework's provenance record notes both findings together "suggest[] the underlying `.docx` corpus itself contains save/export defects independent of anything done during integration."

## Status

Open. This is primarily an editorial/source-fidelity question (whether and how to reconcile the source file) rather than a question of constitutional meaning — but the decision belongs to whoever holds authority over the source corpus, not to this reorganisation.
